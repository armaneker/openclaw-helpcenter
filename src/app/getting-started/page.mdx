# Getting Started

This guide walks you through deploying your first OpenClaw agent and building toward a full multi-agent setup.

## What is OpenClaw?

OpenClaw is an AI agent infrastructure that lets you run multiple fully isolated agents within a single Gateway process. Each agent gets its own workspace, memory, tools, and model assignment — while sharing one server process for simple operations.

## Quick Start

### 1. Install OpenClaw

Install the OpenClaw CLI on your server:

```bash
openclaw onboard --install-daemon
```

This installs OpenClaw as a systemd daemon with auto-restart and logging.

### 2. Create Your First Agent

```bash
openclaw agents add coordinator
```

The wizard will prompt you through configuration options including model selection and auth setup.

### 3. Set Agent Identity

```bash
openclaw agents set-identity --agent coordinator --name "Coordinator Agent"
```

### 4. Configure the Workspace

Navigate to your agent's workspace directory and create the core files:

```bash
cd ~/.openclaw/workspace-coordinator
```

Create these files:
- **SOUL.md** — Who the agent is (persona, tone, domain expertise)
- **AGENTS.md** — How the agent operates (protocols, rules, workflows)
- **USER.md** — Who the users are and their context
- **TOOLS.md** — Available tools and API references

### 5. Connect a Channel

Bind your agent to a messaging channel (e.g., Slack, Teams):

```bash
openclaw agents bind --agent coordinator --bind slack:main
```

### 6. Start the Gateway

```bash
openclaw gateway --port 18789
```

### 7. Verify

```bash
openclaw doctor
openclaw channels status --probe
```

## Rollout Strategy

<Callout type="warning" title="Start Small">
Start with one agent and add incrementally. Fewer than 10% of teams successfully scale beyond single-agent setups — coordination complexity is the #1 challenge.
</Callout>

| Phase | Duration | Goals |
|-------|----------|-------|
| **Phase 1** | Weeks 1–2 | Single agent, one channel, validate memory and sessions |
| **Phase 2** | Weeks 3–4 | Add a second specialist agent (e.g., engineering) |
| **Phase 3** | Weeks 5–6 | Add remaining agents, build custom skills |
| **Phase 4** | Weeks 7–8 | Enable sub-agent orchestration and automated workflows |
